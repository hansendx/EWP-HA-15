Besteht bereits ein zentrales Repository klont man mit git clone das zentrale Repository in ein locales Repository auf dem eigenen Rechner. Wenn im localen Repository Dateien geändert werden können diese durch git add zur staging area hinzugefügt werden. Die Änderungen in den Dateien in der staging area können dann wiederum mit git commit in einem commit festgehalten werden. Die local festgehaltenen commits werden dann mit git push zum zentralen Repository hochgeladen, wo die Änderungen mit den dortigen Dateien verschmolzen werden. (Sofern es möglich ist die Dateien zu vereinigen und keine Widersprüche auftreten) Falls sich in dem zentralen Repository etwas geändert hat, kann man mit git pull die Änderungen ins locale Repository ziehen. Auch hier kann es dazu kommen, dass widersprüche auftreten, die per hand repariert werden müssen
